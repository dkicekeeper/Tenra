From AIFinanceManager/Services/Transactions/TransactionQueryService.swift (current, problematic):
```swift
@MainActor
class TransactionQueryService: TransactionQueryServiceProtocol {
    // WRONG — nonisolated(unsafe) bypasses actor isolation but does NOT prevent races
    nonisolated(unsafe) private static let dateFormatter = DateFormatters.dateFormatter

    func calculateSummary(...) -> Summary {
        let dateFormatter = Self.dateFormatter  // accessed on MainActor — OK, but declaration is wrong
        ...
    }
}
```

The class is already `@MainActor`. The correct declaration is:
```swift
@MainActor private static let dateFormatter = DateFormatters.dateFormatter
```

This is safe because:
1. `TransactionQueryService` is `@MainActor` — all instance methods run on MainActor
2. A `@MainActor static let` is accessed on MainActor, matching the class isolation
3. No `Task.detached` or background context accesses this formatter

Rule from CLAUDE.md: "Use `@MainActor private static let` for DateFormatters; format strings on
MainActor, pass String (not formatter) to Task.detached."
</interfaces>
