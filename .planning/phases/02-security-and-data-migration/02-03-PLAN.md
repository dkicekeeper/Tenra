<objective>
Create an explicit CoreData mapping model for the v2 → v3 schema migration so that upgrading users do not encounter a crash when the persistent store loader tries to migrate the changed `dateSectionKey` attribute.

Purpose: DATA-01 — the v2 → v3 diff is a single attribute change (`TransactionEntity.dateSectionKey`: `transient=YES` → persistent with `defaultValueString=""`). Lightweight migration handles this in theory, but an explicit mapping model documents the intent, is deterministic, and prevents fallback to "inferred" migration that could fail on older devices or in edge-case store states.

Output: A `.xcmappingmodel` bundle inside the `.xcdatamodeld` package, containing an XML `contents` file that maps all v2 entities to v3 entities. CoreDataStack already has both migration flags set so no Swift code changes are needed.
</objective>

  <action>
Create the mapping model as a raw XML file. CoreData's `.xcmappingmodel` format is a directory containing a single `contents` XML file.

Place the file directly inside the `.xcdatamodeld` bundle. The `xcodebuild` `mapc` step automatically picks up all `.xcmappingmodel` directories inside `.xcdatamodeld`.

Final path:
  AIFinanceManager/CoreData/AIFinanceManager.xcdatamodeld/AIFinanceManager_v2_to_v3.xcmappingmodel/contents

Write the following XML as the `contents` file:

```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<mapping type="com.apple.IDECoreDataModeler.MappingModel">
    <source name="AIFinanceManager v2"/>
    <destination name="AIFinanceManager v3"/>
    <entityMappings>
        <entityMapping name="AccountEntityMapping" sourceEntityName="AccountEntity" destinationEntityName="AccountEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="CategoryAggregateEntityMapping" sourceEntityName="CategoryAggregateEntity" destinationEntityName="CategoryAggregateEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="CategoryRuleEntityMapping" sourceEntityName="CategoryRuleEntity" destinationEntityName="CategoryRuleEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="CategorySubcategoryLinkEntityMapping" sourceEntityName="CategorySubcategoryLinkEntity" destinationEntityName="CategorySubcategoryLinkEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="CustomCategoryEntityMapping" sourceEntityName="CustomCategoryEntity" destinationEntityName="CustomCategoryEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="RecurringOccurrenceEntityMapping" sourceEntityName="RecurringOccurrenceEntity" destinationEntityName="RecurringOccurrenceEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="RecurringSeriesEntityMapping" sourceEntityName="RecurringSeriesEntity" destinationEntityName="RecurringSeriesEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="SubcategoryEntityMapping" sourceEntityName="SubcategoryEntity" destinationEntityName="SubcategoryEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="TransactionEntityMapping" sourceEntityName="TransactionEntity" destinationEntityName="TransactionEntity" type="NSMigrationCopyEntityMigrationPolicy">
            <attributeMappings>
                <attributeMapping name="dateSectionKey" valueExpression="$source.dateSectionKey"/>
            </attributeMappings>
        </entityMapping>
        <entityMapping name="TransactionSubcategoryLinkEntityMapping" sourceEntityName="TransactionSubcategoryLinkEntity" destinationEntityName="TransactionSubcategoryLinkEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
        <entityMapping name="MonthlyAggregateEntityMapping" sourceEntityName="MonthlyAggregateEntity" destinationEntityName="MonthlyAggregateEntity" type="NSMigrationCopyEntityMigrationPolicy"/>
    </entityMappings>
</mapping>
```

Note on nil handling: `$source.dateSectionKey` is the correct NSExpression — the `??` nil-coalescing operator is not valid NSExpression syntax and will cause a `mapc` compiler error. When the source value is nil (because the attribute was transient in v2 and never persisted), CoreData automatically applies the destination attribute's `defaultValueString=""`.

Use the Write tool to create this file. The directory `AIFinanceManager_v2_to_v3.xcmappingmodel/` will be created as part of the path.
