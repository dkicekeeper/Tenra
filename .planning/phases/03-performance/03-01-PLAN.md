<objective>
Add `categoryTotals: [String: Double]` to `PreAggregatedData.build()` inside its existing O(N) loop, then wire `generateSpendingInsights` to use this dictionary for the `.allTime` granularity instead of grouping all transactions.

Purpose: The `.allTime` generator currently calls `Dictionary(grouping: topExpenses, by: { $0.category })` followed by `.map { resolveAmount }` — an O(N) scan over all 19k transactions. Replacing this with an O(1) dictionary lookup eliminates the 307ms bottleneck entirely.

Output: `InsightsService.swift` gains `categoryTotals` field in `PreAggregatedData`; `InsightsService+Spending.swift` uses it in the `topExpenses`/`sortedCategories` computation for the allTime case.
</objective>
