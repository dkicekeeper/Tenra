HOW TO CREATE IN-MEMORY STORE:
The model is loaded from the app bundle. Use `NSPersistentContainer` with `NSInMemoryStoreType`:
```swift
func makeInMemoryContainer() throws -> NSPersistentContainer {
    // Model name must match the .xcdatamodeld bundle name
    let container = NSPersistentContainer(name: "AIFinanceManager")
    let description = NSPersistentStoreDescription()
    description.type = NSInMemoryStoreType
    description.shouldAddStoreAsynchronously = false
    container.persistentStoreDescriptions = [description]
    var loadError: Error? = nil
    container.loadPersistentStores { _, error in
        loadError = error
    }
    if let error = loadError { throw error }
    return container
}
```
The container uses the same NSManagedObjectModel as production — loaded from the app's bundle by the `NSPersistentContainer(name:)` initializer. In test targets, the app bundle is available via `@testable import AIFinanceManager`.

SAVE AND RELOAD PATTERN:
```swift
let context = container.viewContext
// 1. Create entity using the factory
let entity = TransactionEntity.from(transaction, context: context)
// 2. Save — this triggers willSave() → dateSectionKey auto-populated
try context.save()
// 3. Reload via fetch
let request = TransactionEntity.fetchRequest()
request.predicate = NSPredicate(format: "id == %@", transaction.id)
let results = try context.fetch(request)
let reloaded = results.first!
// 4. Convert back and compare
let roundTripped = reloaded.toTransaction()
```

KNOWN GOTCHA: `createdAt` field stores `Date?` on the entity; `Transaction.createdAt` is `TimeInterval`. `toTransaction()` uses `createdAt?.timeIntervalSince1970 ?? Date().timeIntervalSince1970`. Compare with tolerance: `abs(roundTripped.createdAt - transaction.createdAt) < 1.0`.

KNOWN GOTCHA: `convertedAmount == 0` is treated as nil by `toTransaction()`. If you set `convertedAmount = nil` on the `Transaction`, the entity stores `0.0`, and `toTransaction()` returns `nil`. This is correct behavior; verify it in Test E.
